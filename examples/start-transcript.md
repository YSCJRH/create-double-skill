# start transcript

现在的 `start` 不再默认所有人都走同一组固定问题。

first run 的节奏变成：

1. 先选分身用途
2. 再选访谈深度
3. 先拿到第一个 artifact
4. 用一句 correction 修正
5. 如果愿意，再继续细化 2-4 个问题

最常见的几条路径：

- [start-transcript-general.md](start-transcript-general.md)
- [start-transcript-work.md](start-transcript-work.md)
- [start-transcript-self-dialogue.md](start-transcript-self-dialogue.md)
- [start-transcript-external.md](start-transcript-external.md)

如果你只是想先跑起来，最推荐直接试：

```powershell
python scripts/double_builder.py start --slug my-work-double --display-name "工作分身" --use-case work
```
