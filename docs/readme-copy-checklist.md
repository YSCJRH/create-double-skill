# README Copy Checklist

这份清单只服务 README、examples、release 文案，不影响运行逻辑。

## 首屏规则

- 首屏先给命令，再给解释
- 首屏只保留一条主路径
- 首屏先展示结果文件或输出片段，再讲理念
- 首屏不要同时推 4-5 条入口

## 去 AI 腔规则

- 少用“不是 X，不是 Y，而是 Z”这种连续对比开场
- 少用标签堆叠式自夸，例如 `local-first / editable / auditable / correction-friendly`
- 少用“这套流程的重点不是……”这种替系统辩护的句式
- 少用口号式收束句
- 少用抽象价值词，优先写具体动作、具体问题、具体文件

## 更像工具 README 的写法

- 多写“运行后会发生什么”
- 多写“你会得到什么文件”
- 多写“如果不对，下一步怎么改”
- 多写可复制命令
- 多写 before / after，而不是原则堆叠

## 发布前快速自查

- 首页第一屏是否能在 15 秒内看懂
- 首条命令是否复制即用
- transcript 是否和真实 CLI 一致
- 是否先看到结果，再看到理念
- 是否还有明显的产品腔或 AI 腔
